バージョン管理システムはシステム開発の現場では、もはや在って当たり前。
なくてはならないツールです。

SVN

SVN(SubVersion) はオープンソースのバージョン管理システムです。ド定番中のド定番。
エンジニアなら1度は扱ったことがある（はず）。
ちなみにapacheライセンスである。

Gitとよく比較され、SVNは集中型のバージョン管理システムと呼ばれる。

eclipseではSVN向けのプラグインもあり、非常に相性がよい。
ただ、バージョン管理システムとしては老舗なだけあって、Gitと比べるとUIがやや古臭い。

Git

git も定番のバージョン管理システムである。
SVNとの違いは、gitは分散型のバージョン管理であり、ライセンスはGNUとなっている。
分散型は他にもMercurial、Bazaarなどがある。

何と言ってもブランチの作成やソースマージなどが簡単にできることと、
GitHubやGitLabのようなホスティングサービスの存在が非常に便利。

現在も、そして今後もgitに関する知識やノウハウは必要不可欠になるであろうし、
SVNのような集中型を扱うプロジェクトは少しずつ減少していくと思われる。
